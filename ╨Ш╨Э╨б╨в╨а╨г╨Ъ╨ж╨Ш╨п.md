# 🏠 ИПОТЕКА CRM — Next.js Полная инструкция деплоя

## Архитектура проекта

```
crm-nextjs/
├── pages/
│   ├── index.js              ← Весь фронтенд CRM (UI)
│   ├── _app.js               ← Обёртка Next.js
│   └── api/                  ← БЭКЕНД (все API маршруты)
│       ├── auth/login.js     ← POST /api/auth/login
│       ├── clients/
│       │   ├── index.js      ← GET/POST /api/clients
│       │   └── [id].js       ← GET/PUT/DELETE /api/clients/:id
│       ├── managers/
│       │   ├── index.js      ← GET/POST /api/managers
│       │   └── [id].js       ← PUT/DELETE /api/managers/:id
│       ├── users/
│       │   ├── index.js      ← GET/POST /api/users
│       │   └── [id].js       ← PUT/DELETE /api/users/:id
│       ├── pipeline/index.js ← GET/PUT /api/pipeline
│       ├── checklists/index.js← GET/PUT /api/checklists
│       ├── wa/
│       │   ├── webhook.js    ← POST /api/wa/webhook (Green API)
│       │   ├── send.js       ← POST /api/wa/send
│       │   └── chats.js      ← GET /api/wa/chats
│       ├── dashboard/index.js← GET /api/dashboard
│       ├── kpi/index.js      ← GET /api/kpi
│       ├── tasks/index.js    ← GET /api/tasks
│       └── search/index.js   ← GET /api/search
├── lib/
│   ├── supabase.js           ← Supabase клиент (серверный)
│   ├── auth.js               ← JWT авторизация
│   └── api.js                ← Фронтенд fetch-враппер
├── styles/globals.css
├── package.json
├── next.config.js
└── .env.example              ← Шаблон переменных окружения
```

# ШАГ 1 — SUPABASE (10 минут)

1. Зайдите на **supabase.com** → Sign up → New project
2. **Settings → SQL Editor → New query**
3. Откройте файл `schema.sql` → скопируйте всё → вставьте → Run
4. **Settings → API** → скопируйте:
   - `Project URL` → например `https://abcdef.supabase.co`
   - `service_role` ключ (НЕ anon!) → длинная строка

---

# ШАГ 2 — GITHUB (5 минут)

1. Зайдите на **github.com** → New repository
2. Название: `mortgage-crm` → Create
3. Нажмите **"uploading an existing file"**
4. Перетащите ВСЕ файлы из папки `crm-nextjs`
   (все папки: pages, lib, styles + все файлы)
5. Commit changes

---

# ШАГ 3 — VERCEL (5 минут)

1. Зайдите на **vercel.com** → Add New Project
2. Найдите `mortgage-crm` → Import
3. **ВАЖНО перед деплоем** — добавьте переменные окружения:
   В разделе "Environment Variables" добавьте:

```
SUPABASE_URL          = https://abcdef.supabase.co
SUPABASE_SERVICE_KEY  = eyJh...ваш service_role ключ...
JWT_SECRET            = придумайте любую строку от 32 символов
GREEN_API_ID          = оставьте пустым пока не подключите WA
GREEN_API_TOKEN       = оставьте пустым пока не подключите WA
NEXT_PUBLIC_APP_URL   = https://mortgage-crm.vercel.app
```

4. Нажмите **Deploy**
5. Через ~2 минуты получите ссылку: `https://mortgage-crm-xxx.vercel.app`

# ШАГ 5 — WHATSAPP (после основного деплоя)

1. Зайдите на **green-api.com** → Регистрация
2. Создайте инстанцию → отсканируйте QR вашим WhatsApp
3. Скопируйте `idInstance` и `apiTokenInstance`
4. В Vercel → Settings → Environment Variables → добавьте/обновите:
   ```
   GREEN_API_ID    = ваш idInstance
   GREEN_API_TOKEN = ваш apiTokenInstance
   ```
5. В Green API → Settings → Webhook URL:
   ```
   https://ваш-сайт.vercel.app/api/wa/webhook
   ```
6. Включите: Incoming messages ✅
7. Нажмите Save

Теперь когда клиент напишет в WhatsApp →
автоматически создастся лид в CRM!

# КАК РАБОТАЕТ АВТОРИЗАЦИЯ

```
Менеджер вводит логин/пароль
        ↓
POST /api/auth/login
        ↓
Бэкенд проверяет в таблице users
        ↓
Возвращает JWT токен (живёт 7 дней)
        ↓
Фронтенд сохраняет в localStorage
        ↓
Все запросы идут с заголовком: Authorization: Bearer {token}
        ↓
Бэкенд проверяет токен на каждом API маршруте
```

# WHATSAPP — КАК РАБОТАЕТ

```
Клиент пишет в WA → Green API получает → отправляет на:
  POST https://ваш-сайт.vercel.app/api/wa/webhook
        ↓
  Бэкенд (pages/api/wa/webhook.js):
  1. Парсит сообщение (текст/фото/голосовое/документ)
  2. Создаёт запись в wa_chats
  3. Создаёт клиента в таблице clients (этап: new_lead)
  4. Сохраняет сообщение в wa_messages
        ↓
  Менеджер в CRM видит новый чат + лид
        ↓
  Менеджер отвечает из CRM → POST /api/wa/send →
  Green API → сообщение клиенту в WhatsApp
```

Поддерживаемые типы сообщений:
- ✅ Текст
- ✅ Фото (с подписью)
- ✅ Голосовые сообщения
- ✅ Аудио
- ✅ Видео
- ✅ Документы (PDF, Word и т.д.)
